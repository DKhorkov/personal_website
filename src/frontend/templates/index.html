Plants Care Telegram Bot

Plants Care Telegram Bot - это телеграм бот, созданный для ухода и заботы о наших зеленых друзьях: растениях :) На данный момент в боте реализована функция отслеживания полива, но в дальнейшем может быть добавлены функция для мониторинга необзодимости удобрения или пересадки. Текущий тех позволяет создавать сценарий полива, который объединяет растения по группе, давать описание этому сценарию, интервал полива, указывать дату последнего полива. А уже далее можно добавить цветок в созданный ранее сценарий, указав название растения, дополнительное описание, а также прикрепить фото. Как сценарии полива, так и данные о самих растениях можно удалять и редактировать.

Технологии, использованные при создании проекта:

  • PyTelegramBotAPI
  • SQLAlchemy
  • Docker Compose
  • SQLite
  • Telebot-Calendar
  • Requests

Natalia Toys Webstore

Natalia Toys Webstore - это полноценный небольшой интернет магазин (backend часть) для продажи плюшевых игрушек авторской ручной работы. Данный магазин написан на Django-framework, имеет каталог, корзину, функцию оплаты (тестовой) через Яндекс-кассу, личный кабинет, историю заказов. Регистрация производится с помощью emailи переходу по подтверждающей ссылке либо через соцсети: Вконтакте или Гугл. При оформлении заказа он сохраняется в список заказов полььзователя, где также есть информация о том, олпачен заказ или нет. После оплаты заказа пользователю отправляется чек, а владельцу (или потенциальному отправщику от его имени) приходит на почту сообщение о том, что определенынй пользователь (с указанием адреса и контактов) купил игрушку/игрушки в n-ном количестве и ждет их получения.

Технологии, использованные при создании проекта:

  • Django
  • Yookassa
  • SQLite
  • HTML
  • Jinja2
  • CSS

Task Manager

Task Manager - это веб-приложение для создания и отслеживания любых задач. Данное веб-приложение было написано на современном асинхронном фреймворке FastApi. Задачи можно создавать, менять их статус (открывать, закрывать, переоткрывать) и удалять. Регистрация производится по email и паролю: Если пользователь зарегистрирован, идет проверка валидности введенного пароля (происходит хэширование пароля и сравнение с хэшом в базе данных, созданным в момент регистрации). Если пароль валидный, то создается токен для данного юзера, сохраняется в БД и добавляется в cookies ответа RedirectResponse, который перенаправляет пользователя на страницу с его задачами. Срок жизни токена 2 недели.

Технологии, использованные при создании проекта:

  • FastApi
  • Cookies
  • Async SQLAlchemy
  • PyTest
  • SQLite
  • Hashlib
  • Logging
  • Pydantic
  • HTML
  • Jinja2
  • CSS

Guess The Number Game

Guess The Number Game - это игра для двоих по локальной сети, где первый игрок загадывает число в определенном диапазоне, а второй пытается его отгадать в рамках заданного количества попыток. По окончании раунда (когда закончились попытки или было угано число), происходит изменение счета, его вывод для игроков и смена позиций (первый игрок становится вторым, а второй первым). Данная игра имеет как консольную версию, так и версию с GUI. Во второй, которая является основной, на старте можно либо создать сервер (с указанием адреса и порта для создания, а также с диапазоном числа и количеством попыток на то, чтобы его угадать), либо подключиться к нему. При подключении создается игровое чат-окно, в котором юзеры и взаимодействуют друг с другом.

Технологии, использованные при создании проекта:

  • Socket
  • Tkinker
  • Pickle

Flappy Bird Game

Flappy Bird Game - это всем известная мобильная игра, переписанная на компьютер с помощью библиотеки PyGame. Главной особенностью данного "ремейка" является предварительный графический интерфейс, в котором можно установить количество жизней, цвета, уровень сложности, музыку, под котороую пользователь хочет играть, а также размер непосредственного окна с игрой. Еще одной особенностью является персональный рекорд и предыдущие конфигурации: пользователь всегда соревнуется с собой на том же уровне сложности и с тем же количеством жизней.

Технологии, использованные при создании проекта:

  • PyGame
  • Pillow
  • Tkinker
  • SQLite
  • Pickle
